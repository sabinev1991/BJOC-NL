Importeer Groet Speler in een ander programma

Vaak zul je een bepaalt soort gebruik van code aantreffen die je al weleens hebt gemaakt in een ander programma.

Op deze pagina, ga je jouw groet speler blok kopieeren in je Raad-het-getal spel, zodat je het ook daar kunt gebruiken.

    This project is from Unit 1 Lab 2 Page 4: Learning Player Names.
  1. Open je H1L2-GroetSpeler project, run het een paar keer, en her-bekijk de code zodat je je weer herinnert hoe deze werkt. (Op het moment dat je wat namen hebt in de spelers lijst, test het dan met namen die wel en namen die niet op de lijst staan.)
  2. Fix alle problemen die je tegenkomt bij groet speler. Als het werkt naar behoren, exporteer het. (Instructies hieronder.)

    Exporteren van blokken

    1. Kies "Exporteer blok..." uit het Snap! file menu ().
    2. Selecteer alleen de blokken die jij wilt exporteren (in dit geval, alleen groet speler), en klik "OK." Een XML document wordt nu gedownload.
      3. De Snap! Handleiding vertelt je meer over exporteren en importeren.
  3. En dan nu, open je H2L1-Raad-het-getal project, en importeer je groet speler blok.

    Importeren van blokken

    1. Kies "Importeer..." uit het Snap! file menu ().
    2. Zoek en vind je XML bestand die je geexporteerd hebt (misschien op je desktop in je 'download' map) en klik Openen.
    3. Vind de blokken die je wilt. Deze zullen aan het eind van je palette verschijnen.
    4. Klik het geimporteerde blok om ervoor te zorgen dat alles het netjes doet. (In dit voorbeeld, zul je waarschijnlijk een error tegenkomen nadat je een naam hebt ingevoerd.)
  4. Look at the code for greet player. Can you determine what's wrong?

Debugging afhankelijken

Soms als je een script pakt uit een programma waarin je het gebouwd hebt en gebruik voor een ander programma, gaat het stuk. Het kan zijn dat het script afhankelijk is op een speciaal blok dat niet is gedefinieerd in het nieuwe programma, of het kan zijn (zoals in dit geval) dat het script afhankelijk is van een globale variabele dat nog niet is gedefinieerd in het nieuwe programma.

Om dit probleem op te lossen, kun je een blok importeren, en de globale variabele definieeren in je nieuwe programma. Afhankelijken zijn een reden om script variabelen te gebruiken in plaats van globale variabelen. Omdat deze gedefenieerd zijn binnen hun scripts, zul je ze niet kwijtraken als je je script verplaatst naar een andere plek.

  1. Maak de ontbrekende globale variabele, speler lijst, en zorg dat je programma het initieert als een lege lijst.
    Om een lege lijst aan te maken, klik op de linker pijl op de lijst-blok, zodat er geen plaats is voor input.

    2. Als de speler lijst is geiniteerd, zal het spel elke keer wanneer het wordt opgestart, geen enkele naam onthouden. Dus, je hebt een ander moment nodig waarop het spel weer wordt gestart, om op die manier de speler lijst te initieren. Om een voorbeeld te geven, zou je de groene vlag kunnen gebruikten om de lijst te initieren en de spatiebalk om het Raad-het-getal spel te starten.

  2. Test en Debug. Je programma zou moeten:
    • begroeten van de speler en zijn/haar naam onthouden;
    • beginnen met spelen van het Raad-het-getal spel met een nieuw geheim getal.
    Fix problemen met je code.
    Is er een probleem met je variabele?
    • Check de variabele initiatie (waar het is gemaakt): Is dit waar de code hoort te zijn? Doet de code waarvoor het is bedoeld?
    • Check waar de variabele gebruikt wordt: Gebruik je de variabele die je in gedachte had? Gebruik je het op de manier zoals jij het hebt bedacht?
